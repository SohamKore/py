  • Can add, remove, and update elements
  • Maintains order (index-based access)
  • Allows duplicates
  • Slower for searching (O(n) time complexity)
my_list = [10, 20, 30, 40]

ğŸ“Œ 2. Tuple (tuple) - Immutable, Ordered, Allows Duplicates

  • Maintains order
  • Faster than lists (immutable)
  • Allows duplicates
  • Cannot add, remove, or update elements
my_tuple = (10, 20, 30, 40)
# my_tuple[1] = 25  # âŒ TypeError: 'tuple' object is immutable

ğŸ“Œ 3. Set (set) - Mutable, Unordered, Unique Elements

  • No duplicate elements
  • Faster for searching (O(1) average)
  • Supports set operations (union, intersection, etc.)
  • Unordered (no index-based access)
my_set = {10, 20, 30, 40, 10}  # Duplicate ignored
my_set.add(50)
print(my_set)  # {10, 20, 30, 40, 50}

ğŸ“Œ 4. Frozen Set (frozenset) - Immutable, Unordered, Unique

  • Same as set but immutable
  • Cannot add or remove elements
my_frozenset = frozenset([10, 20, 30, 40])
# my_frozenset.add(50)  # âŒ AttributeError: 'frozenset' object is immutable

ğŸ“Œ 5. Dictionary (dict) - Mutable, Ordered (since Python 3.7), Key-Value Pairs

  • Stores data in key-value pairs
  • Fast lookups (O(1) average)
  • Keys must be unique
my_dict = {"name": "Om", "age": 20, "city": "Pune"}
my_dict["age"] = 21
print(my_dict)  # {'name': 'Om', 'age': 21, 'city': 'Pune'}

ğŸ“Œ 6. Range (range) - Immutable, Generates Numbers on Demand

  • Efficient for looping, does not store all numbers
  • Cannot modify once created

print(list(r))  # [1, 3, 5, 7, 9]
